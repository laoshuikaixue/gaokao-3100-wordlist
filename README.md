# 高考英语 3100 词表

本项目以教育部《普通高中英语课程标准（2017年版2025年修订）》附录2为最高依据，整理维护高中英语3100词表、2025差异版、2026正式版及配套乱序/默写资料。

乱序脚本只调整词条的排列顺序，不修改所选版本中的单词、音标、词性、释义、派生词和星号标记。

## 项目特点

- 按完整词条进行乱序，避免拆散跨行或跨页的释义。
- 保留原文件开头的【编写说明】。
- 保留必修词 `*`、选择性必修词 `**` 等原始标记。
- 使用楷体中文和 Times New Roman 英文、音标排版。
- 在 PDF 末页标注 `Powered By LaoShui`。
- 对照教育部2025修订课标核验词头、星级和完整性。
- 使用固定随机种子，默认情况下可重复生成相同的排列结果。
- 生成前后会计算词条内容摘要，确认乱序过程没有修改词条内容。

## 目录说明

| 目录或文件 | 说明 |
| --- | --- |
| `原版/` | 原版 PDF、原版乱序 PDF、乱序生成脚本及 `默写/` 成品 |
| `校正版/` | 蓝色2025差异版、黑色2026版、校正明细、审校报告、2026乱序 PDF、生成脚本及 `默写/` 成品 |
| `build_corrected_3100.py` | 2025差异版与2026版的唯一生成脚本 |

## 环境要求

- Python 3.10 或更高版本
- [PyMuPDF](https://pymupdf.readthedocs.io/)
- [ReportLab](https://www.reportlab.com/opensource/)

安装 Python 依赖：

```bash
pip install pymupdf reportlab
```

当前脚本默认使用 Windows 自带字体：

- `C:\Windows\Fonts\STKAITI.TTF`
- `C:\Windows\Fonts\STSONG.TTF`
- `C:\Windows\Fonts\times.ttf`
- `C:\Windows\Fonts\timesbd.ttf`

如果在 Linux 或 macOS 上运行，请在 `make_shuffled_3100.py` 中将字体路径替换为本机可用的中文楷体、宋体和 Times 系列字体。

## 生成乱序版

脚本会根据输入文件名中是否含有“校正版”，自动切换封面和编写说明。在对应版本目录中运行：

```bash
python ./make_shuffled_3100.py \
  "2025届高考英语《新课程标准》3100词总表.pdf" \
  "2025届高考英语新课程标准3100词总表_乱序版.pdf"
```

PowerShell 示例：

```powershell
python .\make_shuffled_3100.py `
  '.\2025届高考英语《新课程标准》3100词总表.pdf' `
  '.\2025届高考英语新课程标准3100词总表_乱序版.pdf'
```

2026版目录中的命令：

```powershell
python .\make_shuffled_3100.py `
  '.\2025届高考英语《新课程标准》3100词总表（2026版）.pdf' `
  '.\2025届高考英语《新课程标准》3100词总表（2026版）_乱序版.pdf'
```

## 自定义乱序结果

脚本默认使用固定随机种子：

```text
LaoShui-gaokao-3100-wordlist-2026-07-20
```

传入不同的 `--seed` 即可生成另一套乱序结果：

```bash
python make_shuffled_3100.py input.pdf output.pdf --seed "my-random-seed"
```

相同的原始 PDF 和随机种子会生成相同的词条顺序，方便协作、复现和版本管理。

## 生成默写版

原版和校正版的 `默写/` 目录分别包含以下三种模式：

- 混合：部分题目给英文写中文，部分题目给中文写英文。
- 中写英：所有题目均给出中文释义。
- 英写中：所有题目均给出英文单词。

每种模式均生成正序题册、乱序题册和乱序答案。脚本会自动填充页面可用空间，并使用固定种子保证乱序题册与答案顺序一致。

在相应的 `默写/` 目录中运行：

```powershell
python .\make_dictation_3100.py '..\2025届高考英语《新课程标准》3100词总表.pdf' '.'
```

正式版运行时将输入文件替换为上一级目录中的2026版 PDF；输出文件统一使用“2026版”命名。

## 实现方式

脚本会依次完成以下处理：

1. 从原始 PDF 中提取正文的视觉文本行。
2. 识别普通词条、带星号词条、无斜线音标缩写和跨页释义。
3. 将同一词条的全部内容组合为不可拆分的词条块。
4. 使用指定随机种子打乱词条块顺序。
5. 重新排版并生成带封面、页码、署名和来源说明的 PDF。
6. 对乱序前后的词条内容计算 SHA-256 摘要，确保内容集合一致。

## 关于词条数量

“3100词”沿用教育部课标名称。教育部附录2可提取 `3093` 条展示行，其中部分斜线/括号变体合并计数。2026版将原版错误并入相邻词条的项目拆开，并补入18个未独立列出的课标词，当前生成 `3108` 个可独立检索的词条块。

脚本不会删除未被单独识别为新词条的文本；这些内容仍会保留在所属的完整词条块中。
